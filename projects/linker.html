Linker

This is a course project for my course of operating system taught by Professor Hubertus Franke at NYU courant.

In general, a linker takes individually compiled code/object modules and creates a single executable by resolving external symbol references (e.g. variables and functions) and module relative addressing by assigning global addresses after placing the modulesâ€™ object code at global addresses.

In this project, I implemented a two-pass linker. In the first pass, I find out some warnings and errors and output the symbol table. In the second pass, I dealt with some warnings and errors and then print out the memory map.

The code is wrote in C++.

My steps to finish such a task is:

  • Construct a getToken function that can read from the input file toke by token ("1","E","1004" are called tokens).
  • Since the linker has two part, so I divide my task into part 1 and part 2.
  • Basically, I read the whole file module by module.
  • At first, I use a correct input without any warnings and errors to test my code to make sure my code can deal with the most correct situation. After that, I started to deal with the warnings and error could happen and write my own test cases to test my code.
